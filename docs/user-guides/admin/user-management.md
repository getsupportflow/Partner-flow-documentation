# User Management

User Management in PartnerFlow allows administrators to define who can access the platform, what they can see, and which actions they are authorised to perform. This ensures that services, inventory, and financial data are secure while keeping workflows collaborative and efficient.

## Creating and Managing Users

Navigate to **Settings → Users** to manage your team.

To add a new user:
1. Click **Create New User**
2. Enter the user's name and email address
3. Assign a role
4. Save — the user receives an invitation email

You can edit a user's role, suspend their access, or remove them from the account at any time.

## Roles and Permissions

Roles control what each user can see and do. PartnerFlow includes standard roles out of the box:

| Role | Typical access level |
|---|---|
| **Admin** | Full access to all settings, services, users, billing, and data |
| **Management** | Full visibility across services and reports; limited settings access |
| **Support** | View and update service records; no access to billing or service settings |
| **Technician** | Access only to assigned services and linked inventory |

### Granular Permission Controls

Each role can be configured with specific permissions covering:

- **Service Records** — view, create, update, transition, assign
- **Service Builder** — create, edit, and publish services
- **Inventory** — view listings, manage stock, create purchase orders
- **Accounting** — view estimates and invoices, approve, mark as paid
- **Notifications and Watchlists** — receive alerts on case changes
- **Documentation** — view, upload, or edit service documents
- **Contracts** — view, sign, generate

## Custom Access Levels

Tailor access for individual users based on organisational needs:

- A **Technician** may see only the services they are assigned to and the inventory linked to those services
- A **Management** user has full visibility across all services and can access analytics and reporting
- A **Support** user can communicate with customers and update records but cannot modify the service flow or access financial data

To create a custom role, go to **Settings → Roles**, define the permission set, and assign it to users as needed.

## Audit and Activity Logs

All user activity is logged in the system. Navigate to **Settings → Audit Log** to review:

- Who made changes to a service record
- Stage transitions and the user who triggered them
- Configuration changes (service edits, user additions, role changes)
- Login activity

Audit logs support accountability, compliance, and troubleshooting. Service creators can review provider activity, and service providers can evaluate their own team's performance.

## Benefits

| Benefit | Detail |
|---|---|
| **Security** | Control who can access sensitive data and prevent unauthorised actions |
| **Flexibility** | Adapt roles and permissions to the needs of your organisation and partners |
| **Transparency** | Clear oversight of all user activity within the system |
| **Efficiency** | Only the right users can update services, inventory, or billing — reducing errors |
| **Collaboration** | Users work together safely within their permitted scope |
