# Service Documentation

The Documentation feature provides a centralised way to store and access all relevant files associated with a service. Everything can be organised and linked directly to the service record for quick reference by internal teams and service providers.

:::note
Documentation is for internal use and service providers only. Customers with portal access cannot see these files.
:::

## Typical Uses

- Attaching warranty guidelines, photos, or technical reports
- Uploading quality checklists for use at different stages
- Providing technical repair or installation guides
- Storing compliance documents for audits
- Sharing manufacturer-approved training materials

## Creating Folders

Organise related documents under logical categories — for example, Inspection Reports, Training Materials, QA Records, or Compliance Docs.

To create a folder:
1. Open the **Documentation** tab in the service editor
2. Click **Create Folder**
3. Enter a name and save

## Creating Documents

Within a folder, you can upload or link different file types:

- Text notes and written guides
- Images and diagrams
- Video links (YouTube, Vimeo, or internal links)
- External documents (PDF, DOCX)

To add a document:
1. Open the folder where you want to add the file
2. Click **Create Document**
3. Upload the file or paste a link
4. Add a title and optional description

## Permissions

Permissions can be customised per document or folder:

- **View only** — users can read but not modify
- **Edit** — users can update content
- **Upload** — users can add new files

This ensures sensitive documents (e.g. legal agreements, audit records) are protected while operational guides remain openly accessible.

## Where Documents Appear

Documents are linked directly to the service and appear in every service record created from that service. Teams and partners can access them from the record view without navigating back to the service settings.
